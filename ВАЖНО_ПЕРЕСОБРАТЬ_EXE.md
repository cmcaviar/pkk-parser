Старый EXE файл был собран **ДО** внесения исправлений для Windows. Поэтому он не содержит новый код с:

## Инструкция по пересборке EXE на Windows

### Вариант 1: Быстрый тест (без сборки EXE)

Если у вас на Windows установлен Python, можно протестировать исправления сразу:

1. Скопируйте обновлённые файлы на Windows:
   - `api_client.py` (с исправлениями)
   - `gui_app.py`
   - `config.py`
   - `models.py`
   - `excel_export.py`
   - `test_windows_fix.py` (новый тестовый скрипт)

2. Запустите тест:
   ```cmd
   python test_windows_fix.py
   ```

3. Проверьте вывод:
   - Должны увидеть логи с "Windows: SSL проверка отключена"
   - Должны увидеть "Используемый User-Agent: Mozilla/5.0 (Windows NT 10.0..."
   - Данные должны получиться

4. Если тест успешен, запустите GUI:
   ```cmd
   python gui_app.py
   ```

---

### Вариант 2: Полная пересборка EXE

#### Шаг 1: Подготовка

1. Скопируйте ВСЕ файлы проекта на Windows

2. Откройте командную строку (cmd) в папке проекта

3. Убедитесь что PyInstaller установлен:
   ```cmd
   pip install pyinstaller
   ```

#### Шаг 2: Сборка

Запустите скрипт сборки:
```cmd
python build_exe.py
```

Выберите вариант **1** (Собрать .exe файл)

#### Шаг 3: Тестирование

1. После сборки найдите файл: `dist/nspd_parser.exe`

2. Запустите его

3. **Включите "Dev логи"** в интерфейсе (чекбокс внизу)

4. Введите тестовый номер: `77:01:0001003:1926`

5. Нажмите "Старт"

6. Проверьте логи в интерфейсе - вы должны увидеть:
   ```
   [DEBUG] Платформа: Windows 10
   [DEBUG] Используемый User-Agent: Mozilla/5.0 (Windows NT 10.0...
   [DEBUG] Windows: SSL проверка отключена
   [DEBUG] Windows: Настроен HTTPAdapter с retry стратегией
   [DEBUG] Инициализация сессии (попытка 1/3)...
   [DEBUG] Статус ответа: 200
   [DEBUG] Content-Type: application/json
   [DEBUG] JSON успешно распарсен
   [INFO] Объект найден: 77:01:0001003:1926
   ```

7. Если увидели эти логи - **исправления работают!**

## Диагностика если проблема остаётся

Если после пересборки данные всё ещё не получаются:

1. **Проверьте логи** - теперь они будут детальные

2. **Проверьте папку `debug_responses/`** - там будут сохранены проблемные ответы API

3. **Экспортируйте логи**:
   - В GUI нажмите кнопку "📄 Экспорт логов"
   - Сохраните файл
   - Откройте его и найдите секцию с ошибкой

4. **Проверьте что видно в логах**:
   - Если `Content-Type: text/html` вместо `application/json` - API блокирует запрос
   - Если `ERROR - Ответ не содержит поле 'data'` - неправильная структура ответа
   - Если `SSL ошибка` - проблемы с сертификатами (должно быть исправлено)
