## Ключевые особенности

### 🔄 API-First с Selenium Fallback
- Первичная попытка через API (быстро, эффективно)
- Автоматическое переключение на Selenium при блокировке
- Самоподписанный сертификат обработан (`verify_ssl=False`)

### 🛡️ Устойчивость к ошибкам
- **Retry**: 3 попытки с экспоненциальной задержкой (5, 10, 15 сек)
- **HTTP коды**: обработка 404, 429, 500
- **Continue on error**: сбой одного номера не останавливает процесс
- **Stop flag**: graceful shutdown через файл `stop.flag`

### 📊 Excel интеграция
- Чтение из листа "Входные данные"
- Автоматическая запись в лист "Результат"
- Форматирование: заголовки, автоширина, freeze panes
- **Множественные объекты**: если на участке 5 зданий → 5 строк

### 📝 Подробное логирование
- Файл `parser.log` с ротацией (max 10 MB)
- Уровни: DEBUG, INFO, WARNING, ERROR
- Timestamp + модуль + сообщение
- Просмотр через VBA макрос `ViewLogs()`

### ⚙️ Гибкая конфигурация
- Все настройки в `config.py`
- Настройка колонок Excel
- Таймауты и retry
- User-Agent и заголовки

## Сценарии использования

### Сценарий 1: Командная строка
```bash

python main.py data.xlsx

# С созданием отдельного файла результатов
python main.py data.xlsx --create-new-file

python main.py data.xlsx --log-level DEBUG
```

## Требуемая адаптация ⚠️

Перед первым использованием **ОБЯЗАТЕЛЬНО** адаптировать:

### 1. API endpoints
```python
# config.py
search_endpoint = "/api/РЕАЛЬНЫЙ_ENDPOINT"
details_endpoint = "/api/РЕАЛЬНЫЙ_ENDPOINT"
```

### 2. JSON парсинг
```python
# api_client.py → parse_parcel_data()
parcel.address = data.get('РЕАЛЬНОЕ_ПОЛЕ_АДРЕСА')
parcel.area = data.get('РЕАЛЬНОЕ_ПОЛЕ_ПЛОЩАДИ')
```

### 3. Selenium селекторы
```python
# selenium_fallback.py → search_cadastral_number()
search_input = By.CSS_SELECTOR, "input.РЕАЛЬНЫЙ_КЛАСС"
search_button = By.CSS_SELECTOR, "button.РЕАЛЬНЫЙ_КЛАСС"
```

**См. подробное руководство в `ADAPTATION_GUIDE.md`**

## Типичные проблемы

| Проблема | Решение |
|----------|---------|
| "Лист не найден" | Создать лист **"Входные данные"** (точное название!) |
| "Колонка не найдена" | Заголовок **"Кадастровый номер объекта недвижимости"** |
| "Парсер не найден" (VBA) | Положить .exe/файлы рядом с Excel файлом |

| "Selenium не запускается" | Установить Google Chrome |
| "Объект не найден" | Проверить кадастровый номер, доступность сайта |
| "Неправильные данные" | **Адаптировать API/селекторы!** (см. ADAPTATION_GUIDE.md) |

## Производительность

### API режим
- ~2-3 сек на кадастровый номер
- 1000 номеров ≈ 40-50 минут

### Selenium режим
- ~5-7 сек на кадастровый номер
- 1000 номеров ≈ 90-120 минут

### Рекомендации
- Использовать API режим (по умолчанию)
- Selenium только как fallback
- Для больших объемов: запуск ночью или split на batch'и

## Безопасность

✅ **Обработано:**
- SSL self-signed сертификаты
- User-Agent spoofing
- Задержки между запросами (1-2 сек)
- Retry с экспоненциальной задержкой

⚠️ **Рекомендации:**
- Не превышать 1000 запросов/час (риск блокировки IP)
- Использовать задержки в config.py
- При блокировке - увеличить `retry_delay`
