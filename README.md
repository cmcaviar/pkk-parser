# Парсер НСПД (https://nspd.gov.ru)

Полнофункциональный парсер для получения данных об объектах недвижимости по кадастровым номерам из Национальной системы пространственных данных (НСПД).

## Возможности

- **API-First подход** с автоматическим переключением на Selenium при необходимости
- **Пакетная обработка** кадастровых номеров из Excel
- **Устойчивость к ошибкам**: retry механизм, обработка 404/429/500
- **Детальное логирование** всех операций
- **Интеграция с Excel**: запуск по кнопке, автоматическая запись результатов
- **Гибкая конфигурация** через config.py
- **Сборка в .exe** для работы без установки Python

## Архитектура

```
├── main.py                  # Точка входа
├── config.py               # Конфигурация (таймауты, retry, колонки)
├── models.py               # Модели данных (Parcel, RealtyObject)
├── api_client.py           # HTTP клиент с retry логикой
├── selenium_fallback.py    # Selenium парсер (fallback)
├── parser_core.py          # Оркестрация API + Selenium
├── excel_io.py             # Чтение/запись Excel
├── logger_setup.py         # Настройка логирования
├── requirements.txt        # Зависимости Python
├── build_exe.bat           # Сборка в .exe (Windows)
├── run_hidden.vbs          # Скрытый запуск без консоли
└── excel_macro.bas         # VBA макрос для Excel
```

## Требования

- **Python 3.11+** (для запуска из исходников)
- **Windows** (тестировалось на Windows 10/11)
- **Excel** (для использования VBA интеграции)
- **Chrome** (для Selenium fallback)

## Использование

### 1. Подготовка Excel файла

Создайте Excel файл со структурой:

**Лист "Входные данные":**

| Кадастровый номер объекта недвижимости |
|----------------------------------------|
| 77:01:0001001:1                        |
| 77:01:0001001:2                        |
| ...                                    |

### 2. Запуск парсера

#### Из командной строки

```bash
# С использованием .exe
nspd_parser.exe data.xlsx

# С использованием Python
python main.py data.xlsx

# С дополнительными опциями
python main.py data.xlsx --output results.xlsx --log-level DEBUG
```

#### Из Excel (VBA макрос)

1. Откройте Excel файл с данными
2. `Alt + F11` (открыть редактор VBA)
3. `Insert` → `Module`
4. Вставьте код из `excel_macro.bas`
5. Создайте кнопку на листе:
   - Вкладка "Разработчик" → Вставить → Кнопка
   - Привяжите к макросу `RunNSPDParser`
6. Нажмите кнопку для запуска

### 3. Результаты

Результаты автоматически сохраняются в:
- **Лист "Результат"** в исходном файле (по умолчанию)
- Или в новый файл `result_YYYY-MM-DD.xlsx` (опция `--create-new-file`)

#### Структура результатов:

**Данные участка:**
- Кадастровый номер (вход)
- Вид объекта (участок)
- Вид участка
- Адрес (участок)
- Площадь (участок)
- ВРИ
- Форма собственности (участок)
- Кадастровая стоимость (участок)

**Данные объектов:**
- Вид объекта (объект)
- Кадастровый номер (объект)
- Назначение
- Площадь (объект)
- Форма собственности (объект)
- Кадастровая стоимость (объект)
- Удельный показатель
- Этажи
- Подземные этажи
- Материал стен
- Завершение
- Ввод в эксплуатацию
- Культурное наследие
- Статус обработки
- Ошибка

## Аргументы командной строки

```bash
python main.py <excel_file> [опции]

Позиционные аргументы:
  excel_file            Путь к Excel файлу с кадастровыми номерами

Опциональные аргументы:
  -h, --help            Показать справку
  -o, --output OUTPUT   Путь к выходному файлу
  --selenium            Использовать только Selenium (без API)
  --log-level LEVEL     Уровень логирования (DEBUG, INFO, WARNING, ERROR)
  --log-file FILE       Файл для логов (по умолчанию: parser.log)
  --no-console          Не выводить логи в консоль
  --create-new-file     Создать новый файл вместо добавления листа
```

## Конфигурация

Основные настройки в `config.py`:

```python

timeout = 30
max_retries = 3
retry_delay = 5.0
request_delay_min = 1.0
request_delay_max = 2.0

# Selenium настройки
headless = True
page_load_timeout = 30

# Excel настройки
input_sheet = "Входные данные"
output_sheet = "Результат"
```

## Логика работы

1. **Чтение данных** из Excel
2. **Для каждого кадастрового номера:**
   - Попытка получения через API
   - При неудаче → переключение на Selenium
   - Извлечение данных участка
   - Извлечение объектов на участке
   - Запись результатов (несколько строк если объектов много)
3. **Обработка ошибок** без остановки процесса
4. **Сохранение результатов** в Excel

## Устойчивость к ошибкам

- **Retry механизм**: до 3 попыток для каждого запроса
- **Экспоненциальная задержка**: 5, 10, 15 секунд
- **Обработка HTTP кодов**: 404, 429, 500
- **SSL bypass**: работа с самоподписанными сертификатами
- **Continue on error**: один сбой не останавливает весь процесс
- **Stop flag**: создайте файл `stop.flag` для остановки

## Известные ограничения

1. **Структура API**: Реальные endpoint'ы НСПД могут отличаться - необходима адаптация селекторов в `api_client.py`
2. **Selenium селекторы**: Требуется настройка под реальную HTML структуру сайта в `selenium_fallback.py`
3. **Rate limiting**: При большом количестве запросов возможна блокировка - используйте задержки
4. **Chrome**: Selenium требует установленный Chrome

## Адаптация под реальный API

После анализа сайта НСПD через DevTools необходимо обновить:

### В `api_client.py`:
```python
# Реальные endpoints
search_endpoint = "/api/actual/endpoint"

# Структура JSON ответа
def parse_parcel_data(self, data: Dict[str, Any], cadastral_number: str):
    # Адаптировать под реальную структуру
    parcel.object_type = data.get('realFieldName')
    ...
```

### В `selenium_fallback.py`:
```python
# Реальные CSS селекторы
search_input = By.CSS_SELECTOR, "input.real-class"
parcel.address = safe_get_text(".real-address-selector")
...
```

## Troubleshooting

### "Парсер не найден"
- Убедитесь, что `nspd_parser.exe` или `main.py` находится в директории Excel файла
- Или укажите полный путь в VBA макросе

### "Лист 'Входные данные' не найден"
- Создайте лист с точным названием "Входные данные"
- Или измените название в `config.py`: `input_sheet = "Ваше название"`

### "Selenium не запускается"
- Установите Chrome
- Проверьте наличие `webdriver-manager` в зависимостях
- Запустите с `--log-level DEBUG` для деталей

### "SSL ошибка"
- Сайт использует самоподписанный сертификат
- Убедитесь, что `verify_ssl = False` в `config.py`

### "Объект не найден"
- Проверьте правильность кадастрового номера
- Проверьте доступность сайта НСПД
- Проверьте логи для деталей
