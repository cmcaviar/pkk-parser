# 🔧 Сборка .exe файла на Windows

Инструкция для разработчика: как собрать standalone .exe приложение из исходников.

---

## 📋 Требования

1. **Windows** (7, 8, 10, 11 - любая версия)
2. **Python 3.8+** (должен быть установлен)
3. **Все зависимости проекта**

---

## ⚙️ Подготовка окружения (один раз)

### Шаг 1: Установите Python

Если Python ещё не установлен:

1. Скачайте Python с https://www.python.org/downloads/
2. При установке **обязательно** поставьте галочку **"Add Python to PATH"**
3. Проверьте установку:
   ```cmd
   python --version
   ```

### Шаг 2: Установите зависимости

Откройте **Command Prompt** (cmd) или **PowerShell** в папке проекта:

```cmd
cd C:\путь\к\pkk-parser

REM Установите все зависимости
pip install -r requirements.txt

REM Установите PyInstaller для сборки
pip install pyinstaller
```

---

## 🚀 Сборка .exe файла

### Способ 1: Автоматическая сборка (рекомендуется)

Просто запустите скрипт сборки:

```cmd
python build_exe.py
```

Выберите опцию **"1"** (Собрать .exe файл) или **"3"** (Собрать и очистить).

Скрипт автоматически:
- ✅ Проверит наличие всех файлов
- ✅ Соберёт .exe через PyInstaller
- ✅ Упакует все зависимости
- ✅ Создаст готовый файл в папке `dist/`

### Способ 2: Ручная сборка

Если предпочитаете контроль, выполните команду вручную:

```cmd
pyinstaller ^
  --name=nspd_parser ^
  --onefile ^
  --windowed ^
  --clean ^
  --hidden-import=requests ^
  --hidden-import=openpyxl ^
  --hidden-import=urllib3 ^
  gui_app.py
```

**Параметры:**
- `--name=nspd_parser` - имя итогового .exe файла
- `--onefile` - собрать всё в один файл (не папку)
- `--windowed` - GUI приложение без консоли
- `--clean` - очистить временные файлы перед сборкой
- `--hidden-import=...` - явно включить модули (иногда PyInstaller их не находит)

## 📁 Результат сборки

После сборки структура проекта:

```
pkk-parser/
├── gui_app.py          # Исходник
├── build_exe.py        # Скрипт сборки
├── ...                 # Другие файлы
│
├── build/              # Временные файлы (можно удалить)
├── dist/
│   └── nspd_parser.exe # 🎉 ГОТОВЫЙ .EXE ФАЙЛ!
└── nspd_parser.spec    # Конфигурация PyInstaller (можно удалить)
```

**Готовый файл:** `dist/nspd_parser.exe`

**Размер:** ~50-70 МБ (включает Python + все библиотеки + код)

## 🐛 Решение проблем

### Проблема 1: "PyInstaller не найден"

**Ошибка:**
```
'pyinstaller' is not recognized as an internal or external command
```

**Решение:**
```cmd
pip install pyinstaller
```

### Проблема 2: "ModuleNotFoundError" при запуске .exe

**Причина:** PyInstaller не нашёл какой-то модуль автоматически.

**Решение:** Добавьте модуль в `build_exe.py`:

```python
pyinstaller_args = [
    # ...
    '--hidden-import=имя_модуля',  # Добавьте эту строку
]
```

### Проблема 3: .exe файл слишком большой

**Нормальный размер:** 50-70 МБ.

**Если больше 100 МБ:**
- Проверьте что используете `--onefile`
- PyInstaller мог случайно включить лишние библиотеки

**Решение:** Явно исключите ненужное:

```python
pyinstaller_args = [
    # ...
    '--exclude-module=numpy',      # Если numpy не используется
    '--exclude-module=matplotlib',  # Если matplotlib не используется
]
```

### Проблема 4: Антивирус блокирует .exe

**Причина:** Антивирусы иногда считают упакованные PyInstaller'ом файлы подозрительными.

**Решение:**
1. Добавьте файл в исключения антивируса
2. Или подпишите .exe цифровой подписью (для production)

### Проблема 5: Консольное окно мелькает при запуске

**Причина:** Не указан параметр `--windowed`.

**Решение:** Убедитесь что в `build_exe.py` есть `'--windowed'`.

### Проблема 6: Долгий запуск приложения

**Причина:** Это нормально для PyInstaller. Первый запуск ~5-10 секунд.

**Почему:** .exe распаковывает Python runtime во временную папку.

**Решение:** Это неизбежно для `--onefile` режима. Для ускорения можно использовать `--onedir`, но тогда будет папка с множеством файлов.

## 📦 Распространение

### Вариант 1: Просто .exe файл

Скопируйте `dist/nspd_parser.exe` куда нужно. Всё!

**Преимущества:**
- ✅ Один файл
- ✅ Не требует установки
- ✅ Работает везде

**Недостатки:**
- ⚠️ Большой размер (~50-70 МБ)
- ⚠️ Медленный первый запуск

### Вариант 2: Установщик (опционально)

Для профессионального распространения можно создать установщик:

**Инструменты:**
- **Inno Setup** (бесплатно) - https://jrsoftware.org/isinfo.php
- **NSIS** (бесплатно) - https://nsis.sourceforge.io/
- **InstallForge** (бесплатно) - https://installforge.net/

**Преимущества установщика:**
- ✅ Создаёт ярлык на рабочем столе
- ✅ Добавляет в меню "Пуск"
- ✅ Выглядит профессионально
- ✅ Есть деинсталлятор

## 🔄 Обновление приложения

Если вы изменили код:

1. Внесите изменения в исходники (`gui_app.py`, `api_client.py`, и т.д.)
2. Запустите сборку заново:
   ```cmd
   python build_exe.py
   ```
3. Новый .exe будет в `dist/`
4. Замените старый файл новым

## 💡 Советы

### Ускорение сборки

При разработке постоянно пересобирать долго. Используйте:

```cmd
# Тестируйте без сборки:
python gui_app.py

# Собирайте только когда готово для релиза
```

### Версионирование

Добавьте версию в имя файла для отслеживания:

```python
# В build_exe.py измените:
app_name = "nspd_parser_v1.0"
```

Или добавьте номер версии в интерфейс:

```python
# В gui_app.py:
self.root.title("НСПД Парсер v1.0 - Кадастровые данные")
```

### Логирование ошибок

Если пользователи сообщают об ошибках, добавьте логирование в файл:

```python
# В gui_app.py, в начале:
import logging

logging.basicConfig(
    filename='nspd_parser.log',
    level=logging.DEBUG,
    format='%(asctime)s - %(levelname)s - %(message)s'
)
```
