# 🚨 КРИТИЧНО: Найдена проблема с прокси на Windows!

## Реальная причина проблемы

Windows пытается использовать **несуществующий прокси** `127.0.0.1:2080`, из-за чего все запросы к API блокируются.

### Типичная ошибка в логах:

```
Ошибка инициализации сессии (попытка 1): HTTPSConnectionPool(host='nspd.gov.ru', port=443):
Max retries exceeded with url: /map (Caused by ProxyError('Unable to connect to proxy',
NewConnectionError("HTTPSConnection(host='127.0.0.1', port=2080): Failed to establish a new connection:
[WinError 10061] Подключение не установлено, т.к. конечный компьютер отверг запрос на подключение")))
```

---

## Исправление ВНЕСЕНО!

В файле `api_client.py` добавлено **принудительное отключение прокси для Windows**:

```python
# КРИТИЧНО: Отключаем прокси на Windows
self.session.trust_env = False  # Игнорировать системные настройки прокси
self.session.proxies = {
    'http': None,
    'https': None,
}
```

## Техническая информация

### Почему возникла эта проблема?

Windows (или какая-то программа на компьютере) настроила системный прокси `127.0.0.1:2080`, который:
- Не запущен/не существует
- Блокирует все HTTPS запросы из Python

Библиотека `requests` по умолчанию использует системные настройки прокси (`trust_env=True`), поэтому она пыталась подключиться через этот мёртвый прокси.

### Решение:

```python
self.session.trust_env = False  # Игнорировать системные настройки
self.session.proxies = {'http': None, 'https': None}  # Явно отключить прокси
```

Теперь Python будет подключаться напрямую, минуя прокси.
