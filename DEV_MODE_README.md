# 🔧 Режим разработчика (Dev Mode)

## Описание

В GUI приложение добавлен **режим разработчика** с подробным логированием всех операций. Это полезно для:
- 🐛 Отладки проблем
- 📊 Анализа производительности
- 🔍 Детального изучения работы API
- 📝 Создания отчётов об ошибках

---

## Возможности

### ✅ Автоматическое логирование

Все логи **автоматически** сохраняются в файл, независимо от режима:
- 📁 Папка: `logs/`
- 📄 Формат файла: `nspd_parser_YYYYMMDD_HHMMSS.log`
- 💾 Кодировка: UTF-8
- 🔒 Уровень: DEBUG (все подробности)

### ✅ Dev режим в интерфейсе

При включении чекбокса **"🔧 Dev логи"**:
- [DEBUG] сообщения отображаются в GUI
- Время выполнения API запросов
- Типы данных объектов
- Детальная информация об объектах
- Полные traceback ошибок

### ✅ Экспорт логов

Кнопка **"📄 Экспорт логов"**:
- Копирует текущий лог-файл
- Предлагает выбрать место сохранения
- Автоматически открывает файл (опционально)

## Использование

### 1. Включить dev режим

1. Запустите приложение: `python gui_app.py`
2. Установите чекбокс **"🔧 Dev логи"**
3. Начните работу

**Что увидите:**
```
🔧 Режим разработчика ВКЛЮЧЕН
[DEBUG] Debug логи будут отображаться в интерфейсе
[DEBUG] Файл логов: logs/nspd_parser_20260331_123456.log
```

### 2. Работать с данными

Все операции будут логироваться детально:

**Обычный режим:**
```
[1/3] 77:05:0001016:22
⏳ Загрузка данных участка...
✅ Участок: Москва, ул. Ленина, д. 1
✅ Объектов найдено: 2
```

**Dev режим:**
```
[1/3] 77:05:0001016:22
[DEBUG] Начало обработки номера 77:05:0001016:22
⏳ Загрузка данных участка...
[DEBUG] Вызов get_full_parcel_info_with_objects(77:05:0001016:22)
[DEBUG] API запрос выполнен за 1.23 сек
[DEBUG] Parcel type: Parcel
[DEBUG] Objects count: 2
✅ Участок: Москва, ул. Ленина, д. 1
✅ Объектов найдено: 2
[DEBUG]   Объект 1: Здание - 77:05:0001016:1001
[DEBUG]   Объект 2: Сооружение - 77:05:0001016:1002
✅ Данные обработаны
[DEBUG] ParseResult создан, статус: Успешно
```

### 3. Экспортировать логи

1. Нажмите **"📄 Экспорт логов"**
2. Выберите место сохранения
3. Логи будут скопированы
4. Выберите "Да" чтобы открыть файл

## Что логируется

### 🔍 При запуске приложения

- Время запуска
- Версия Python
- Рабочая директория

### 🔍 При обработке номеров

- Количество номеров
- Выходной файл
- Время создания API клиента
- Для каждого номера:
  - Начало обработки
  - Вызов API метода
  - Время выполнения запроса
  - Тип полученного объекта Parcel
  - Количество объектов
  - Список объектов с типами
  - Статус ParseResult
  - Ошибки (с полным traceback)

### 🔍 При создании Excel

- Имя файла
- Количество записей
- Время создания

### 🔍 При ошибках

- Тип ошибки
- Сообщение об ошибке
- Полный traceback
- Контекст (какой номер обрабатывался)

## Технические детали

### Настройка логирования (gui_app.py:54-80)

```python
def _setup_logging(self):
    """Настройка системы логирования"""
    # Создаём логгер
    self.logger = logging.getLogger('NSPDParser')
    self.logger.setLevel(logging.DEBUG)

    # Создаём папку для логов если нет
    os.makedirs('logs', exist_ok=True)

    # Файловый обработчик (всегда пишет DEBUG логи)
    log_filename = f"logs/nspd_parser_{datetime.now().strftime('%Y%m%d_%H%M%S')}.log"
    self.log_file = log_filename
    file_handler = logging.FileHandler(log_filename, encoding='utf-8')
    file_handler.setLevel(logging.DEBUG)
    file_formatter = logging.Formatter(
        '%(asctime)s - %(levelname)s - %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S'
    )
    file_handler.setFormatter(file_formatter)
    self.logger.addHandler(file_handler)
```

### Метод _log (gui_app.py:699-731)

Универсальный метод логирования с поддержкой уровней:
```python
self._log("Сообщение", level="INFO")    # Обычное

```

### Метод _log_debug (gui_app.py:733-744)

Отладочные сообщения (показываются только в dev режиме):

## Управление логами

### Автоматическая очистка

Логи НЕ удаляются автоматически. Рекомендуется периодически чистить папку `logs/`:

```bash
# Удалить логи старше 30 дней (Linux/macOS)
find logs/ -name "*.log" -mtime +30 -delete

# Удалить логи старше 30 дней (Windows PowerShell)
Get-ChildItem logs/ -Filter *.log | Where-Object {$_.LastWriteTime -lt (Get-Date).AddDays(-30)} | Remove-Item
```

### Размер логов

Один сеанс работы генерирует ~10-50 KB логов (зависит от количества обрабатываемых номеров).

## Устранение проблем

### Проблема: "Файл логов не найден"

**Причина:** Папка `logs/` не создана или файл удалён

**Решение:**
```bash
mkdir logs
```

### Проблема: Dev логи не отображаются

**Решение:**
1. Убедитесь, что чекбокс "🔧 Dev логи" включён
2. Перезапустите приложение
3. Проверьте файл лога напрямую

### Проблема: Экспорт не работает на Linux

**Решение:**
Установите `xdg-utils`:
```bash
sudo apt-get install xdg-utils
```

## FAQ

**Q: Логи занимают много места?**
A: Нет, один сеанс ~10-50 KB. Но рекомендуется периодически чистить старые логи.

**Q: Можно отключить логирование в файл?**
A: Технически да, но не рекомендуется. Файловые логи всегда полезны для отладки.

**Q: Dev режим замедляет работу?**
A: Минимально (~1-2%), в основном за счёт вывода в GUI.

**Q: Можно посмотреть логи без dev режима?**
A: Да! Все логи всегда сохраняются в файл. Dev режим только показывает DEBUG в GUI.

**Q: Как отправить логи разработчику?**
A: Нажмите "📄 Экспорт логов", сохраните файл и отправьте его.
