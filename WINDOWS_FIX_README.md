# Исправления для Windows

## Проблема
На Windows приложение не получало данные от API НСПД, хотя запросы выполнялись.

## Что было исправлено

### 1. **User-Agent определение по ОС**
- Добавлено автоматическое определение операционной системы
- Для Windows используется корректный Windows User-Agent
- Для macOS и Linux используются соответствующие User-Agent'ы

### 2. **Улучшенная обработка SSL на Windows**
- Явное отключение SSL проверки для Windows (сайт использует самоподписанный сертификат)
- Добавлен HTTPAdapter с retry стратегией для Windows
- Автоматический retry при ошибках соединения

### 3. **Детальное логирование**
- Добавлен вывод полного тела ответа API (первые 1000 символов)
- Логирование Content-Type и Content-Length
- Определение и логирование HTML ответов (когда вместо JSON приходит HTML)
- Сохранение проблемных ответов в папку `debug_responses/` для анализа

### 4. **Улучшенная инициализация сессии**
- Добавлен retry механизм для инициализации (3 попытки)
- Детальное логирование полученных cookies
- Предупреждение, если cookies не получены

### 5. **Диагностический режим**
- Автоматическое сохранение проблемных ответов API в файлы
- Категоризация ошибок (html_error, no_data_field, json_parse_error и т.д.)
- Подробная информация о платформе и времени в debug файлах

## Как протестировать

### На Windows:

1. Запустите тестовый скрипт:
```cmd
python test_windows_fix.py
```

2. Проверьте логи:
- `test_windows_fix.log` - полный лог выполнения
- Консольный вывод покажет результат

3. Если есть проблемы, проверьте:
- Папку `debug_responses/` - там сохранены проблемные ответы API
- Лог файл `test_windows_fix.log` - детальная информация о каждом запросе

### Запуск GUI приложения:

1. Запустите GUI:
```cmd
python gui_app.py
```

2. Включите "Dev логи" в интерфейсе для детальной диагностики

3. Введите тестовый номер: `77:01:0001003:1926`

4. Нажмите "Старт"

### Если проблема остается:

Проверьте логи на наличие:
- `ERROR - API вернул HTML вместо JSON!` - сайт заблокировал запрос
- `ERROR - Ответ не содержит поле 'data'` - неправильная структура ответа
- `WARNING - Cookies не получены` - проблемы с инициализацией сессии
- `ERROR - SSL ошибка` - проблемы с сертификатами

## Debug файлы

При возникновении ошибок автоматически создаются файлы в `debug_responses/`:

```
debug_responses/
  77_01_0001003_1926_html_error_1234567890.txt
  77_01_0001003_1926_no_data_field_1234567891.txt
```

Эти файлы содержат:
- Кадастровый номер
- Тип ошибки
- Дату и время
- Платформу
- Полный текст ответа API
