# Быстрый старт - Парсер НСПД

## За 5 минут до первого запуска

### Шаг 1: Установка зависимостей

```bash
# Перейдите в директорию проекта
cd pkk-parser

# Установите зависимости Python
pip install -r requirements.txt
```

### Шаг 2: Подготовка Excel файла

Создайте файл `data.xlsx` со структурой:

**Лист "Входные данные":**
```
| Кадастровый номер объекта недвижимости |
|----------------------------------------|
| 77:01:0001001:1                        |
| 77:01:0001001:2                        |
```

### Шаг 3: Первый запуск

```bash
# Базовый запуск
python main.py data.xlsx

# С подробными логами
python main.py data.xlsx --log-level DEBUG
```

### Шаг 4: Проверка результатов

- Откройте `data.xlsx`
- Найдите лист **"Результат"**
- Проверьте столбец **"Статус обработки"**
- При ошибках смотрите файл `parser.log`

## Интеграция с Excel (VBA кнопка)

### Шаг 1: Открыть редактор VBA
- Откройте Excel файл
- Нажмите `Alt + F11`

### Шаг 2: Добавить модуль
- Меню `Insert` → `Module`
- Скопируйте весь код из `excel_macro.bas`
- Вставьте в окно модуля

### Шаг 3: Создать кнопку
- Закройте редактор VBA (`Alt + Q`)
- Вкладка **"Разработчик"** → **Вставить** → **Кнопка**
- Нарисуйте кнопку на листе
- Выберите макрос `RunNSPDParser`
- ПКМ на кнопке → **Изменить текст** → "Запустить парсинг"

### Шаг 4: Запуск
- Положите `nspd_parser.exe` или файлы проекта рядом с Excel файлом
- Нажмите кнопку
- Дождитесь сообщения о завершении



## Решение проблем

### "Лист не найден"
➜ Создайте лист с именем **"Входные данные"** (точно!)

### "Колонка не найдена"
➜ Заголовок должен быть **"Кадастровый номер объекта недвижимости"**

### "Парсер не найден" (VBA)
➜ Положите `nspd_parser.exe` или файлы проекта в ту же папку, что и Excel файл

### "Selenium ошибка"
➜ Установите Google Chrome

### "SSL Error"
➜ Уже обработано! Проверьте `config.py`: `verify_ssl = False`

## Важно перед использованием!

⚠️ **Адаптация под реальный API НСПД:**

Перед первым использованием необходимо:

1. **Изучить API сайта** через DevTools (F12 → Network)
2. **Обновить endpoints** в `api_client.py`:
   ```python
   search_endpoint = "/api/РЕАЛЬНЫЙ_ENDPOINT"
   ```
3. **Адаптировать парсинг JSON** под реальную структуру ответов
4. **Настроить Selenium селекторы** в `selenium_fallback.py`:
   ```python
   search_input = By.CSS_SELECTOR, "input.РЕАЛЬНЫЙ_КЛАСС"
   ```

Текущий код содержит **примерную структуру** - финальная версия зависит от реального API!

## Структура результата

Если на участке **3 объекта** → **3 строки** в результате:
```
| Кадастровый номер | Адрес участка | ... | КН объекта | Назначение | ... |
|-------------------|---------------|-----|------------|------------|-----|
| 77:01:0001001:1   | ул. Ленина    | ... | 77:...:1001| Жилое      | ... |
| 77:01:0001001:1   | ул. Ленина    | ... | 77:...:1002| Нежилое    | ... |
| 77:01:0001001:1   | ул. Ленина    | ... | 77:...:1003| Жилое      | ... |
```

Если объектов **нет** → **1 строка** с участком:
```
| Кадастровый номер | Адрес участка | ... | КН объекта | Назначение | ... |
|-------------------|---------------|-----|------------|------------|-----|
| 77:01:0001001:2   | ул. Мира      | ... | Нет данных | Нет данных | ... |
```
