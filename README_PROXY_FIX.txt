================================================================================
                  🚨 КРИТИЧНО: ПРОБЛЕМА С ПРОКСИ НА WINDOWS!
================================================================================

НАЙДЕНА РЕАЛЬНАЯ ПРИЧИНА:

  Windows пытается использовать несуществующий прокси 127.0.0.1:2080
  Это блокирует ВСЕ запросы к API!

ТИПИЧНАЯ ОШИБКА:
  "ProxyError('Unable to connect to proxy'..."
  "[WinError 10061] Подключение не установлено..."

================================================================================

ИСПРАВЛЕНИЕ УЖЕ ВНЕСЕНО!

В файле api_client.py добавлено принудительное отключение прокси:

  self.session.trust_env = False
  self.session.proxies = {'http': None, 'https': None}

================================================================================

ЧТО НУЖНО СДЕЛАТЬ:

1. Скопируйте обновлённый api_client.py на Windows

2. Запустите тест:

   python test_windows_fix.py

3. Проверьте что в логах есть строка:

   [DEBUG] Windows: Прокси отключены (trust_env=False)

   ✅ Если эта строка есть - прокси отключены, всё работает!
   ❌ Если нет - используете старую версию кода!

4. Если тест прошёл успешно, пересоберите EXE:

   python build_exe.py

================================================================================

КАК ПРОВЕРИТЬ ЧТО ВЕРСИЯ НОВАЯ:

Включите "Dev логи" и найдите:

  [DEBUG] Windows: Прокси отключены (trust_env=False)

Если эта строка есть - версия правильная!

================================================================================

АЛЬТЕРНАТИВНОЕ РЕШЕНИЕ (если не помогло):

Удалите системные переменные прокси:

  1. Win + Pause → Дополнительные параметры → Переменные среды
  2. Удалите HTTP_PROXY и HTTPS_PROXY если они есть

Или в командной строке:

  set HTTP_PROXY=
  set HTTPS_PROXY=
  python test_windows_fix.py
